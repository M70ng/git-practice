# 今日習ったこと：`git checkout -b` とは何か

## 結論
`git checkout -b` は **「新しいブランチを作成して、同時にそのブランチへ移動する」** ための Git コマンド。

---

## 何をしているコマンドか

```bash
git checkout -b ブランチ名
````

この1行で、以下の **2つの操作を同時に実行**している。

1. 新しいブランチを作成する

   ```bash
   git branch ブランチ名
   ```

2. 作成したブランチに切り替える

   ```bash
   git checkout ブランチ名
   ```

つまり `git checkout -b` は、この2操作のショートカットである。

---

## なぜこのコマンドが必要か

Gitは **作業を分岐（ブランチ）で管理するツール**。

* main / develop を汚さずに作業できる
* 失敗してもブランチごと簡単に捨てられる
* レビューやマージがしやすくなる

`git checkout -b` は
**「ここから別の作業・実験を始める」ことを明示する操作**。

---

## 使うべきタイミング

### 使うべき場面

* 新機能を実装するとき
* バグ修正をするとき
* 試しにコードを書いてみるとき
* Issue / 課題ごとに作業するとき

```bash
git checkout -b feature-login
```

### 使わなくてよい場面

* 既存ブランチに移動するだけのとき
* コードを読むだけのとき

```bash
git checkout main
```

---

## 使わないと起きやすい問題

* main ブランチで直接作業してしまう
* コミット履歴が混ざる
* 「これは別案だった」と後から気づく
* 履歴が読めなくなる
* チーム開発でトラブルになりやすい

**ブランチを作らない = 作業や思考を分離していない** 状態。

---

## 最近のGitでの書き方（補足）

`checkout` は多機能すぎるため、最近は役割別コマンドが推奨されている。

```bash
git switch -c ブランチ名
```

これは `git checkout -b` と **意味・動作は完全に同じ**。

---

## 本質的な理解

* ブランチ = 作業・思考の分岐
* `git checkout -b` = 考えを切り替えるための宣言

コードを安全に書くためだけでなく、
**思考を整理するための操作**である。
